### Цель

Реализовать ядро движка рендеринга текста (Headless Architecture).

Ядро вычисляет геометрию и виртуализирует видимую область,
оставаясь независимым от UI-фреймворка.

### Ограничения

- Swift Embedded.
- Без Foundation в core module.
- Zero-dependency подход: любые сторонние пакеты согласовывать отдельно.
- Swift Embedded экспериментальный, поэтому любые сомнительные API
  нужно помечать как требующие compile verification.
- Public API ядра не должен зависеть от Foundation-типов.
- Ядро должно быть пригодно для компиляции под iOS и WASM без изменений в source code.

### Критерии успеха

- Стабильный scroll performance на документах 100k+ строк / >10 MB.
- В дизайн-спецификации нужно превратить “60 FPS” в измеримый headless budget:
  например p95/p99 latency для пересчёта viewport.
- Память, принадлежащая layout/virtualization core, не должна расти линейно
  от размера документа.
- Документ может храниться вне ядра через абстракцию provider/source.
- Строгая виртуализация: расчёт только для видимого viewport + buffer/overscan.
- Компилируется без изменений под iOS и WASM.
- Регрессионные бенчмарки блокируют merge при деградации производительности.
